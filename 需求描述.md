# 📜 TRISMEGISTUS 项目工程简报 (Project Truth)

## 一、 项目背景与初衷 (Project Context)
*   **起源**: 本项目旨在重构 FGO（Fate/Grand Order）玩家的工具体验[cite: 1]。传统的工具软件（如 **Chaldea**）虽然功能强大，但依赖于复杂的图形用户界面（GUI）和多重筛选逻辑[cite: 1]。
*   **愿景**: 利用大语言模型（LLM）的意图识别能力，将工具转化为基于自然语言（LUI）的智能助手 —— **TRISMEGISTUS（翠玉录）**。
*   **进化**: 
    *   **Old Way**: 打开 App -> 选择从者列表 -> 点击筛选 -> 勾选 NP 充能 -> 输入 30[cite: 1]。
    *   **New Way (TRISMEGISTUS)**: 用户输入：“帮我找一下 30 自充的从者有哪些。” -> AI 直接返回渲染好的从者卡片[cite: 1]。
*   **核心理念**: **Vibe Coding**。通过自然语言驱动代码生产，强调开发者与 AI 之间的逻辑共鸣，而非繁琐的手动编码。

## 四、 架构选型与哲学 (Engineering Principles)

1.  **移动端 Native 优先，逻辑下沉**:
    *   **表现层**: 使用 **SwiftUI** + **SwiftData** 构建专业、极简的 iOS 原生界面[cite: 1]。
    *   **逻辑层**: 将复杂的 FGO 数据筛选下沉至微服务（Dart AOT 或 Python），通过强类型接口为端侧提供支持[cite: 1]。

2.  **协议约束 (Strict Protocol)**:
    *   **JSON Only**: 严格拒绝 YAML。所有配置、Prompt、端云数据交换必须遵循 **JSON Schema**[cite: 1]。
    *   **理由**: 确保 AI 生成的数据流具有可预测性和结构稳定性。

3.  **合规性设计 (Compliance)**:
    *   **协议**: 遵循 **CC-BY-NC-SA-4.0** (非商业性使用、相同方式共享)[cite: 1]。
    *   **声明**: 在项目显著位置注明：“数据及部分逻辑源自开源项目 Chaldea”[cite: 1]。

## 五   、 首个任务：30% NP 自充筛选器
**开发目标**: 实现一个能从 `servants.json` 中准确识别出“在第10级拥有 30% 自充效果”的从者提取器。

*   **输入示例**: `"QUERY_CHARGE": 30`
*   **预期输出**: 包含从者 `id`、`name`、`rarity`、`class` 的结构化 JSON 列表。
*   **参考逻辑**: 遍历 `servant.skills` -> 寻找 `funcType == 'gainNp'` -> 检查 `svals[9].value == 3000` (或对应 FGO 存储格式)[cite: 1]。
