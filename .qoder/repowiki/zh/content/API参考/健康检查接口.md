## 简介

Laplace 项目提供了一个简单的健康检查接口，用于服务可用性检测和状态监控。该接口采用最小化设计原则，确保在各种部署环境中都能可靠运行。

该健康检查接口位于 `/api/health` 端点，是一个轻量级的 GET 请求处理器，返回标准化的 JSON 响应格式，包含服务状态和标识信息。

## 接口规范

### 基本信息
- **HTTP 方法**: GET
- **端点路径**: `/api/health`
- **请求格式**: 无参数
- **响应格式**: JSON
- **认证**: 不需要

### 请求示例
```bash
curl -X GET http://localhost:8000/api/health
```

### 响应示例
```json
{
  "status": "ok",
  "service": "laplace"
}
```

## 使用场景

### 负载均衡器配置

#### Nginx 反向代理
```nginx
upstream laplace_backend {
    server 127.0.0.1:8001;
    server 127.0.0.1:8002;
    server 127.0.0.1:8003;
}

server {
    listen 80;
    
    location / {
        proxy_pass http://laplace_backend;
        proxy_next_upstream error timeout invalid_header http_500 http_502 http_503;
        proxy_next_upstream_tries 0;
    }
    
    location /api/health {
        access_log off;
        return 200 '{"status":"ok","service":"laplace"}';
        add_header Content-Type application/json;
    }
}
```

#### HAProxy 配置
```haproxy
frontend http_front
    bind *:80
    mode http
    
    acl is_health path_beg /api/health
    use_backend health_backend if is_health
    
    default_backend laplace_backend

backend health_backend
    mode http
    balance roundrobin
    server health_server 127.0.0.1:8000 check

backend laplace_backend
    mode http
    balance roundrobin
    option forwardfor
    server app1 127.0.0.1:8001 check
    server app2 127.0.0.1:8002 check
    server app3 127.0.0.1:8003 check
```

### 监控系统集成

#### Prometheus Exporter
```python
from prometheus_client import Counter, Histogram
import requests

health_checks = Counter('laplace_health_checks_total', 'Total health checks')
health_check_failures = Counter('laplace_health_check_failures_total', 'Health check failures')

def monitor_health():
    try:
        response = requests.get('http://localhost:8000/api/health')
        health_checks.inc()
        if response.status_code == 200:
            return True
        else:
            health_check_failures.inc()
            return False
    except Exception as e:
        health_check_failures.inc()
        return False
```

## 故障排除指南

### 常见问题诊断

#### 1. 500 Internal Server Error
**症状**: 健康检查返回 500 错误

**可能原因**:
- 应用程序启动失败
- 数据库连接异常
- 配置文件缺失

**解决步骤**:
1. 检查应用程序日志
2. 验证数据库连接
3. 确认配置文件完整性

#### 2. 404 Not Found
**症状**: 返回 404 错误

**可能原因**:
- 路径拼写错误
- 服务未正确部署

**解决步骤**:
1. 验证端点路径
2. 检查服务部署状态
3. 确认防火墙设置

#### 3. 连接超时
**症状**: 请求超时

**可能原因**:
- 端口未开放
- 网络配置问题

**解决步骤**:
1. 检查端口监听状态
2. 验证网络连通性
3. 确认安全组规则
